GSB : Gestion de rapport

Le contexte proposé est celui du laboratoire pharmaceutique GSB. J'ai pu entreprendre ce projet avec 2 personnes le but étant de mettre en place un certain nombre de service définit par un cahier des charge. Le tâche qui m'a été confiée a été de développer un outil de gestion de rapports de visites destiné aux visiteurs médicaux (présentation de nouveaux médicaments à des médecins).

Mes missions

  • Affichage de tous les rapports

  • Visualisation des rapports

  • Formulaire de création de rapport

Langages

ORM

Autres outils

Affichage de tous les rapports

Le projet débute à partir d'une base de données déjà fournie. Ensuite, j'en ai extrai les données pour afficher tous les rapports dans un tableau grâce à l'outil d'infrastructure d'interface utilisateur pour la création d'applications de bureau Windows, Windows Forms.

Visualisation des rapports

Ensuite, un utilisateur peut cliquer sur le bouton visualiser pour afficher le rapport en détails. Travail que j'ai effectué avec Windows Form.

Formulaire de création et de modification de rapport

Enfin, la dernière tâche développée fut de permettre la création et la modification de rapport. Les données sont ensuite insérées ou mises à jour dans la base de données à l'aide de l'entity framework, outil qui manipule la base de données depuis le code. Ainsi, l'utilisateur peut créer un rapport via l'interface.
